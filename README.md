# Wahapedia WH40k GitHub Mirror

Automated hourly sync of Wahapedia WH40k data (official CSV export + Core Rules page parsing) into GitHub, with a static web UI for datasheets and rules.

## What Is Included

- `scripts/fetch_wh40k.py`
  - Downloads official export files from `https://wahapedia.ru/wh40k10ed/`
  - Includes `Datasheets_models_cost.csv` for unit costs
  - Builds `docs/data/core_rules.json` by parsing the Core Rules page
  - Writes only changed files
- `.github/workflows/sync-wahapedia.yml`
  - Runs hourly (`0 * * * *`)
  - Supports manual run from Actions tab
- `docs/`
  - `index.html`: datasheet viewer UI
  - `rules.html`: Core Rules reader UI
  - `data/index.json`: latest sync metadata generated by the script

## Quick Start

1. Push this project to your GitHub repository.
2. Enable GitHub Pages:
   - `Settings` -> `Pages`
   - `Build and deployment` -> `Source: Deploy from a branch`
   - `Branch: main`, folder `/docs`
3. Run the workflow manually:
   - `Actions` -> `Sync Wahapedia WH40k` -> `Run workflow`
4. Open your Pages URL:
   - `https://<user>.github.io/<repo>/`

## Local Sync

```bash
python3 scripts/fetch_wh40k.py
```

## Commit-Only-On-Change Behavior

The workflow runs the sync script, then checks git diff:

- if no changes are detected, it exits without commit;
- if `docs/data` changed, it commits and pushes updated data.

## Notes and Limits

- There is no official standalone Core Rules CSV in the export. Core Rules are parsed from:
  - `https://wahapedia.ru/wh40k10ed/the-rules/core-rules/`
